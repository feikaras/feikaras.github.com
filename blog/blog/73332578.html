Dell 1501 Slackware 12地狱游记 （安装笔记）

       其实乌鸦的slackware 12dvd盘我07年7月份就刻好了。因为当时安装了下，很好很强大，在启动安装程序的内核的时候就在cpu检测的地方挂掉了，用非smp的内核倒是可以进。这样怎么行?双核才用一个核心？乌鸦当时贪图方便，使用了ubuntu系统。虽然bug 多了点，但是好歹还是能用吧，设备都驱动起来了。
       现在寒假玩完了超白痴的opensuse系统，那个yast可以给你搞定绝大多数的设置。搞得我自己都快白痴掉了，他到底改了什么，一点都不知道。为了麻烦麻烦自己（明显找抽），准备装那个slackware12的系统。
       google了一下，原来1501有那么多的问题，首先，ati显示卡，linux下用ati的卡感觉有点找抽。其次， Broadcom Corporation BCM94311MCG wlan mini-PC这个有名的公司的无线网卡。Broadcom一不提供linux驱动，二也不公开硬件规格。这驱动么。。。最后，是ATI的SB600南桥，有pci bug. 最新的BIOS里面看来还是未修正，导致安装系统的时候会需要pci=nomsi或者acpi=off这样的内核参数。。不得不提到的是那东西的SATA控制器。

安装部分

1.插入slackware 12 DVD开始引导，启动的时候他会让你选择内核，这时候选择hugesmp.s记得加上参 数acpi=off,否则会引导不能。

2.按照屏幕提示安装，安装软件包的时候建议大家使用expert专家模式，自己好好得选。不然装完了会发现怎么装了那么多没用的软件。（特别是新闻阅读器，多余的内核，kde的几个播放器，还有多余的桌面环境）我这里不安装LILO,因为已经有grub了。

3.软件包装好了，开始设置下grub,加入slackware的项目，如果还是hugesmp记得还是要跟上参数acpi=off。

4.启动slackware,配置吧，下面才是真的头疼的地方。我抓狂好几次了。

   (3) 内核编译，为了重新打开acpi,以及优化内核
标题所说的地狱就是这里。我在这一步反复摸索，试行错误（-___-japanese)
反反复复得弄了近10小时。。。。
下面GOGOGO,下载刚刚发布不久的Linux kernel 2.6.24 from kernel.org
记得别下成patch。有兴趣的朋友多多折腾。。。。

tar xvf linux-2.6.24.tar.bz2 -C /usr/src
cd /usr/src/linux-2.6.24
make menuconfig

下面加载slackware内核的原有设置,在/boot/config

在Processor type and features里面
最关键的设置Processor family
Timer frequency
Preemption Model
原来这里的设置都是倾向服务器的。
在主菜单的Power management options里面打开
Hibernation
对acpi和cpu frequency scaling作点调整好了
去掉没用的。－ －
下面就是去Networking里面裁减
什么IrDA bluetooth都可以关了
Networking options里面去掉自己不可能用的东西
不去也没啥－ －
说起减肥的话，Device Drivers里面的东西才是大头
去了parallel port support
IEEE 1394
Fusion MPT
以及其他驱动。注意这里有几个东西要注意，
一个是SCSI的adaptec的一堆驱动（我死在这里好多次了，如开头所说）
一个不确定的是Raid，最好别动
I20不确定，最好别动
（上面两个都是在启动的dmesg里面看到的字样）
network方面有很多能去掉

(5) 杀千刀的wireless lan ,无线网卡
这张破卡的驱动折腾了乌鸦我一下午，还让乌鸦误以为内核编错了。
好吧，让我们搞掉他。。
b43 - Linux Wireless
1501的网卡是bcm4311 rev 1的，属于支持范围。
要用b43 (openSuse是bcm43xx,但是这次貌似是b43,都用也没坏处)
http://bu3sch.de/b43/fwcutter/b43-fwcutter-009.tar.bz2
http://downloads.openwrt.org/sources/broadcom-wl-4.80.53.0.tar.bz2

tar xvf b43-fwcutter-009.tar.bz2
cd b43-fwcutter-009
make
make install
安装firmware了
bcm43xx-fwcutter -w /lib/firmware broadcom-wl-4.80.53.0/kmod/wl_apsta_mimo.o
好了，下面重启，用iwconfig多半能看到张网卡(被这张卡折腾了一下午，弄的没脾气了）
